# 理解卡片创作指南

本文档说明如何为「问得出来」撰写一张**合格**的理解训练卡。

## 一张卡究竟在练什么？

先写清一句话 **cognitiveGoal**，例如：

- ✅「辨认标题的情绪设计，再决定要不要信」
- ❌「理解震惊体标题」（太模糊，像在讲知识点）

卡片必须训练**一种可观察的认知动作**，而不是把词典释义背一遍。

## 创作流程（推荐）

1. **选场景**：这张卡主要发生在哪类生活？（见 `ContentScenario`）
2. **定能力**：主要练 1–2 项，辅以 1 项相关能力（共 ≤3 项 `targetSkills`）
3. **定难度**：按五级标准选 `level`，不要按年级硬套
4. **写情境**：`contextStory` 要有具体人物、冲突、选择（100–150 字）
5. **写误解**：`commonMisunderstanding` 写学生真的会怎么想
6. **设计 steps**：3–5 步，从浅到深，每步只考一个点
7. **反思与迁移**：`reflectionQuestion` + `transferTask`
8. **三层提问**：`finalPrompts` 要可拿去问 AI
9. **总结**：`summary` 说清「练会了什么」，不说教

## 字段示例（节选）

```typescript
defineCard({
  id: "net-clickbait-title",
  title: "标题里的「震惊」",
  originalText: "「震惊！某校居然……」——标题常在情绪上推你一把…",
  scenarios: ["网络信息"],
  level: 2,
  targetSkills: ["情绪与语气", "观点与证据", "提问能力"],
  cognitiveGoal: "识别标题的情绪设计，再决定要不要点、信不信",
  commonMisunderstanding: "把情绪强度当成事实强度",
  contextStory: "班级群转了一条「震惊！教育局秘密……」，你点进去发现是卖课广告…",
  steps: [
    step("这类标题主要在利用什么？", ["严谨数据", "好奇愤怒或恐惧", …], 1, "震惊体核心是情绪触发。"),
    // …
  ],
  reflectionQuestion: "我最近转发过的「震惊」标题，证据在哪里？",
  transferTask: "把同一套问题用到短视频封面文案上。",
  finalPrompts: {
    basic: "这条标题试图让我产生什么情绪？",
    deeper: "若去掉情绪词，还剩什么可验证的事实？",
    applied: "我设计三个问题向发布者或搜索核验。",
  },
  summary: "你在练习：先拆标题的情绪设计，再追证据。",
});
```

完整类型见 `types/card.ts`，工厂函数见 `lib/card-helpers.ts`。

## 互动题设计原则

| 原则 | 说明 |
|------|------|
| 一步一点 | 每题只测一个判断，不要复合题 |
| 选项同质 | 四个选项长度、语气相近，避免「一眼假」 |
| 错项要像真错 | 来自真实误解，不是无意义干扰项 |
| 解释要说人话 | `explanation` 告诉「为什么」，不嘲讽 |
| 即时反馈 | 对：巩固关系；错：提示「可能只看到了字面」类引导 |

## 难度怎么选？

| Level | 适合的材料 |
|-------|------------|
| 1 | 词义、基本句意、规则字面 |
| 2 | 因果/转折/比较、题干限定词 |
| 3 | 需要代入校园/同伴/家庭情境 |
| 4 | 前提、利益、例外、隐性成本、伦理 |
| 5 | 迁移 + 自主设计问题、与 AI 协作 |

## 禁忌（常见不合格卡）

1. **纯释义卡**：只有「这句话意思是…」没有认知动作。  
2. **说教卡**：满篇「你应该」「必须」没有互动与情境。  
3. **答案泄露卡**：题干或选项里已写明正确答案。  
4. **道德审判卡**：只分好人坏人，不练他人视角或条件。  
5. **AI 代写卡**：示范「帮我写完」而不是「帮我思考」。  
6. **场景空洞卡**：`contextStory` 只有「小明上学」没有冲突。  
7. **能力堆砌卡**：`targetSkills` 贴满 6 项，实际只练了一个点。  
8. **经典泛滥**：经典短句占比过高，忽视网络、AI、消费等现代场景。

## 质量标准（自检清单）

- [ ] `cognitiveGoal` 一句话说清「练什么动作」  
- [ ] `commonMisunderstanding` 对应真实学生错误  
- [ ] `contextStory` 让初中生愿意读下去  
- [ ] 至少 3 步 `steps`，且难度递进  
- [ ] 每步 `explanation` 非空、非重复题干  
- [ ] `reflectionQuestion` 指向用户自身经验  
- [ ] `transferTask` 换了一个新场景  
- [ ] `finalPrompts` 三层由浅入深、可问 AI  
- [ ] `level` 与步骤深度一致  
- [ ] 场景与能力标签与内容匹配  

## 提交与存放

1. 在 `data/cards/` 下放入对应 `batch-*.ts`  
2. 在 `data/cards/index.ts` 中 import 并加入 `understandingCards`  
3. 本地 `npm run build` 通过  
4. 在 `/explore` 预览筛选与训练流  

## 数量规划建议

| 阶段 | 规模 | 说明 |
|------|------|------|
| 样板 | 30 | 已覆盖主要场景与能力组合 |
| 扩展一期 | 100+ | 每场景至少 8–10 张，经典占比 ≤20% |
| 规模期 | 1000+ | 按场景拆目录或 JSON + CI 校验脚本 |
